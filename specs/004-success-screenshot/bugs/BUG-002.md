# Bug Report: Incorrect Status Filter for Stock Adjustment Screenshot

**Type**: Implementation drift
**Severity**: Low
**Feature**: 004-success-screenshot
**Reported**: 2026-07-05
**Status**: Patched
**Patched**: 2026-07-05

## Description
The user noted that the bot currently selects "All" (empty value `""`) for the Status dropdown when searching for the newly created transaction. The user specified that the transaction status should specifically be set to "Approved" (`"A"`).

## Artifact Traceability

### spec.md
- **Affected user story**: User Story 1 - Task Completion Screenshot
- **Affected requirements**: FR-001 (Clarification from BUG-001)
- **Gap identified**: The clarification in BUG-001 did not specify which exact Status filter to use. It was incorrectly assumed that leaving it blank (All) was the safest approach.

### plan.md
- **Affected sections**: Summary
- **Impact**: The plan specified filtering by today's date, but needs to explicitly state filtering by "Approved" status.

### tasks.md
- **Affected tasks**: T003, T004
- **False completions**: T003, T004
- **Missing tasks**: None.

## Root Cause Analysis
Implementation drift: The developer assumed that catching all statuses was the best approach, but the user explicitly requires the "Approved" status to be selected to correctly filter the view.

1. Run `/speckit-bugfix-patch` to update spec artifacts.
