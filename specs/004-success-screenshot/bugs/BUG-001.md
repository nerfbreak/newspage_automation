# Bug Report: Missing Transaction Details in Stock Adjustment Screenshot

**Type**: Implementation drift / Spec gap
**Severity**: Medium
**Feature**: 004-success-screenshot
**Reported**: 2026-07-04
**Status**: Patched
**Patched**: 2026-07-04

## Description
The user noted that the screenshot taken for Stock Adjustment (Auto Compare & Manual Entry) execution captures the final state of the page right after saving, which typically just shows a cleared or generic success form. The user specifically wants the screenshot to show the newly created transaction in the "List" view grid.

## Artifact Traceability

### spec.md
- **Affected user story**: User Story 1 - Task Completion Screenshot
- **Affected requirements**: FR-001
- **Gap identified**: The spec only states "capture a screenshot of the active browser window immediately upon the successful completion" but did not specify that for specific modules (like Stock Adjustment), the bot must navigate to the list view and search for the transaction to capture meaningful proof.

### plan.md
- **Affected sections**: Summary
- **Impact**: Requires additional navigation logic in `run_execution` and `run_execution_manual` prior to capturing the screenshot.

### tasks.md
- **Affected tasks**: T003, T004
- **False completions**: T003, T004
- **Missing tasks**: None (just reopening and fixing the logic).

## Root Cause Analysis
Implementation drift / Spec gap: The original spec lacked the contextual requirement that "success" means visually proving the transaction exists in the list view, not just capturing the blank form after the save button is clicked.

## Recommended Fix
1. Navigate back to List View.
2. Set "Date To" to Today.
3. Set "Status" to All.
4. Click Search.
5. Wait for the transaction row to appear.
6. Capture the screenshot.
