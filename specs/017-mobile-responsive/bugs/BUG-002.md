# Bug Report: Stock Mutation Group Containers Missing

**Type**: Implementation drift
**Severity**: High
**Feature**: 017-mobile-responsive
**Reported**: 2026-07-10
**Status**: Patched
**Patched**: 2026-07-10

## Description

The Stock Mutation upload state and the three column-mapping groups render directly on the blue page background. The individual uploader feedback, selectboxes, and metric cards retain their own borders, but the outer white Neo-Brutalist group containers are missing.

## Artifact Traceability

### spec.md
- **Affected user story**: User Story 1 - Mobile UI Navigation and Interaction
- **Affected requirements**: FR-001 and FR-004
- **Affected success criterion**: SC-003
- **Gap identified**: The specification requires responsive containers and consistent Neo-Brutalist borders/shadows, but does not explicitly assert that Stock Mutation upload and mapping controls retain their outer group cards.

### plan.md
- **Affected sections**: Summary, Technical Context, and Project Structure
- **Impact**: The UI-only plan remains valid, but source-level regression coverage must verify Streamlit container structure in addition to CSS selector presence.

### tasks.md
- **Affected tasks**: T004, T008, and T010
- **False completions**: None. The regression was introduced after the original responsive feature completed.
- **Missing tasks**: Add a failing source-structure regression test, restore the missing Stock Mutation group containers, and run focused/full smoke verification.

## Root Cause Analysis

Commit `1583a2f` removed the `st.container(border=True)` wrapper around the Stock Mutation uploader while addressing nested visual artifacts. Commit `331b312` then removed the remaining marker. The column-mapping dropdowns and summary metrics were also rendered in separate column passes without a shared group container. The existing CSS smoke test only checks for selector strings in `static/style.css`, so it can pass even when the page no longer emits the required container structure.

## Recommended Fix

1. Patch the responsive spec artifacts with explicit Stock Mutation container requirements and regression tasks.
2. Add a failing smoke test that inspects `pages/4_stock_mutation.py` for the required uploader and mapping group wrappers.
3. Restore the uploader group container and one container per mapping column without changing upload parsing, state handling, or automation logic.
4. Run `/speckit-bugfix-verify`, focused smoke tests, the full smoke suite, and Python compilation.
