# Bug Report: Stock Mutation Execution Tables and Progress Bars Misaligned

**Type**: Spec gap
**Severity**: Medium
**Feature**: 017-mobile-responsive
**Reported**: 2026-07-10
**Status**: Patched
**Patched**: 2026-07-10

## Description

In the desktop Stock Mutation execution view, the DEDUCT and ADD tables can render at different heights because Description cells wrap differently after the DEDUCT quantity receives a minus sign. Each progress bar is placed immediately after its own table, so unequal table heights move the progress bars onto different vertical positions.

## Root Cause Analysis

`utils.render_responsive_dataframe()` renders HTML tables whose row heights are content-driven. The DEDUCT and ADD tables are placed in independent Streamlit columns, so the browser cannot share row or wrapper height across them. When wrapping differs between the two tables, their following progress placeholders start at different vertical coordinates. Existing CSS limits table height to 400px but does not reserve the same height for both execution tables.

## Recommended Fix

1. Add a page-scoped marker immediately before the Stock Mutation execution columns.
2. On desktop only, use that marker to give both execution table wrappers the existing 400px viewport height.
3. Keep mobile card rendering unchanged and verify the source-level contract plus the offline smoke suite.
