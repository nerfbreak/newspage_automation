# Bug Report: Terminate Button Restarts Playwright Instead of Stopping (BUG-010)

**Type**: Logic bug
**Severity**: High
**Feature**: 017-mobile-responsive (terminate UI)
**Reported**: 2026-07-10
**Status**: Patched
**Unlock**: Dama (verified)

## Description

Clicking the TERMINATE → CONFIRM button on the Stock Mutation page causes Playwright to restart instead of stopping. The bot re-enters the execution block on rerun.

## Root Cause

`_setup_terminate_button` in `playwright_engine.py` defines `terminate_callback()` which sets:
- `st.session_state.is_bot_running = False`
- `st.session_state.execute_done = False`

But Stock Mutation (`pages/4_stock_mutation.py`) gates execution on `is_mutasi_running` (line 204), NOT `is_bot_running`. After the callback fires:
1. Streamlit reruns the page
2. `is_mutasi_running` is still `True`
3. Line 204: `if st.session_state.is_mutasi_running:` → True → re-enters execution → restarts Playwright

## Recommended Fix

Make `terminate_callback` also reset `is_mutasi_running` (and clear `mutasi_review_df` to prevent data re-use). Since the callback is generic across all pages, use safe `session_state.pop()` or conditional check.
