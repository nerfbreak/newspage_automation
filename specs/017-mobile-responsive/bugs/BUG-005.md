# Bug Report: Stock Mutation Upload Reset Does Not Restore Dropzone

**Type**: Implementation drift
**Severity**: High
**Feature**: 017-mobile-responsive (Stock Mutation UI)
**Reported**: 2026-07-10

## Description

After clicking "Hapus File Upload" on the Stock Mutation page, the file uploader dropzone does not reappear. The yellow "FILE LOADED" box and red delete button disappear, but the page shows an empty area where the uploader should be instead of restoring the drag-and-drop dropzone.

The `_clear_mutasi_upload()` callback (BUG-004 fix) successfully removes the widget key and clears derived state, but the uploader does not visually reset to its default empty dropzone state.

Potential root causes:
1. `st.session_state.pop()` removes the key but Streamlit's widget tree may not fully re-instantiate the file uploader on the same rerun cycle.
2. The CSS `div[data-testid="stFileUploader"] section { display: none !important; }` injected inline while a file was loaded may persist in the Streamlit DOM cache across reruns even after the injection block is skipped.
3. Streamlit Cloud may still be running the old pre-BUG-004 code (line 97 inline assignment) that crashes on click, preventing any visible state change.

## Root Cause Analysis

The `_clear_mutasi_upload()` callback uses `st.session_state.pop("mutasi_file_uploader", None)` which removes the widget-owned key. On the next Streamlit rerun, `st.file_uploader(key="mutasi_file_uploader")` should re-create fresh with no file selected. However, the uploader component may not fully reset if Streamlit's internal widget tree retains stale metadata for the same key path. A more robust approach is to rotate the widget key (increment a counter) so Streamlit treats it as an entirely new widget instance.

## Recommended Fix
1. Add a `mutasi_uploader_key` counter to session state
2. Use `st.file_uploader(key=f"mutasi_file_uploader_{st.session_state.mutasi_uploader_key}")` 
3. In `_clear_mutasi_upload()`, increment the counter instead of popping the key
4. This forces Streamlit to create a brand-new widget instance on rerun
