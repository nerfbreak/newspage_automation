## Description

The mobile responsive specification allowed `use_container_width=True` as a possible Streamlit responsive parameter. This conflicts with Constitution Principle X, which requires the modern `width` parameter and explicitly deprecates `use_container_width`.

### tasks.md

## Root Cause Analysis

The spec reused legacy Streamlit terminology in an assumption even though the plan and task list already followed the newer constitution rule.

## Recommended Fix

1. Patch `spec.md` to remove the deprecated allowance.
2. Add traceability notes to `plan.md` and `tasks.md`.
